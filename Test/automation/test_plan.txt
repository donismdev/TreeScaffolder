# TreeScaffolder v0.8+ 정밀 테스트 계획 (Test Plan)

본 문서는 정식 1.0 릴리즈 전, 시스템의 모든 예외 상황을 검증하기 위한 시나리오를 정의합니다.

## 1. V2 Multipatch 파서 (v2_parser.py)
- [CASE-V2-01] 태그 불일치: @@@FILE_BEGIN은 있으나 END가 없는 경우.
- [CASE-V2-02] 중복 정의: 동일한 파일 경로가 소스 코드 내에 여러 번 등장하는 경우 (마지막 내용 우선 여부).
- [CASE-V2-03] 특수 문자 경로: 경로명에 공백, 한글, 특수 기호(!@#$%^&())가 포함된 경우.
- [CASE-V2-04] 빈 블록: BEGIN/END만 있고 내용이 전혀 없는 경우.
- [CASE-V2-05] 거대 데이터: 블록 하나에 수십 MB의 텍스트가 포함된 경우 (메모리 및 파싱 시간).

## 2. 트리 재구성 로직 (scaffold_core.py)
- [CASE-TREE-01] 비정상 인덴트: 부모보다 2단계 이상 깊은 인덴트가 바로 나오는 경우.
- [CASE-TREE-02] 루트 누락: @ROOT 선언 없이 바로 트리 구조가 시작되는 경우.
- [CASE-TREE-03] 윈도우 대소문자: 트리에는 'folder', 소스 코드에는 'FOLDER'인 경우의 병합 로직.
- [CASE-TREE-04] 정렬 무결성: 파일명이 숫자/기호로 시작할 때 계층 구조가 깨지지 않는지 확인.

## 3. 실행 및 물리 충돌 (scaffold_runner.py)
- [CASE-EXEC-01] 권한 오류: 덮어써야 할 파일이 '읽기 전용(Read-only)'인 경우.
- [CASE-EXEC-02] 타입 충돌: 파일이 있어야 할 자리에 이미 동일 이름의 '폴더'가 존재하는 경우 (반대 포함).
- [CASE-EXEC-03] 루트 소실: 작업 도중 타겟 루트 폴더가 외부에서 삭제된 경우.
- [CASE-EXEC-04] .gitkeep 지능: 폴더 내에 숨김 파일(.DS_Store 등)만 있을 때 .gitkeep 생성 여부.

## 4. 복구 및 데이터 보전 (recovery_ui.py / runner.py)
- [CASE-RECO-01] 인코딩 혼합: UTF-8이 아닌 파일(CP949)을 덮어썼을 때 복구 로그의 안전성.
- [CASE-RECO-02] 복구 로그 손상: 복구 로그 파일의 일부 블록이 깨졌을 때의 파싱 복구력.
