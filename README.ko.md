# TreeScaffolder: Prompt Helper & File System Scaffolder
**Version 0.8.0**

TreeScaffolder는 **LLM 프롬프팅 효율을 극대화**하고, 텍스트 기반의 설계도를 **실제 파일 시스템에 안전하게 반영**하기 위한 도구입니다.

---

## 🚀 주요 기능

*   **작업 보조**: LLM 결과물을 일일이 복사/붙여넣기/저장하는 번거로운 수작업 제거.
*   **빈 구조 생성**: 소스 코드 없이 트리 텍스트만으로 빈 폴더/파일 껍데기 즉시 생성.
*   **시각적 비교**: 디스크의 실제 상태와 계획된 내용을 한눈에 대조하여 실수 방지.
*   **안전 장치**: 드라이런 및 자동 백업(Recovery)을 통한 데이터 보호.
*   **이력 관리**: 모든 작업을 이름(Job Name)과 함께 폴더별로 깔끔하게 정리.

---

## 📖 사용 방법 (Prompt Workflow)

### 1. 소스 코드 입력 (Source-First)
`Source Code` 탭에 LLM으로부터 받은 소스 코드 블록을 붙여넣으세요. 이 단계만으로도 프로젝트의 폴더 구조가 자동으로 파악됩니다.

### 2. 트리 보완 및 분석
추가적인 빈 폴더가 필요하면 `Scaffold Tree` 탭에 작성합니다. 이후 **Compute Diff (`d`)**를 눌러 실제 디스크와 비교합니다.

### 3. 작업 이름 지정 및 검토
분석이 끝나면 **단축키 `4`**를 눌러 작업 이름을 입력합니다. `After View`에서 색상과 체크박스를 확인하며 적용 대상을 최종 검토합니다.

### 4. 실제 적용 및 복구
**Apply Scaffold (`f`)**를 눌러 실제 파일로 만듭니다. 만약 실수로 덮어썼다면 **Recovery (`q`)** 창에서 언제든 되돌릴 수 있습니다.

---

## ⌨️ 핵심 단축키

*   **d**: 계획 계산 (Compute Diff)
*   **4**: 작업 이름 입력창으로 포커스 이동
*   **f**: 스캐폴드 적용 (Apply Scaffold)
*   **q**: 리커버리 창 열기 (Recovery)
*   **r**: 데이터 초기화 (Clear)
*   **e/w**: 이전 폴더 불러오기 / 새 폴더 찾기
*   **Alt**: 단축키 힌트 표시 (누르고 있기)
