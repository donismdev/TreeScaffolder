# 🌲 Tree Scaffolder 사용자 가이드 (User Guide)

"Tree Scaffolder"는 텍스트로 작성된 설계도(트리 구조와 파일 내용)를 기반으로, 컴퓨터에 실제 폴더와 파일을 쉽고 안전하게 만들어주는 도구입니다. 복잡한 파일 구조를 일일이 손으로 만들 필요 없이, 단 몇 번의 클릭만으로 원하는 개발 환경을 빠르게 구성할 수 있습니다. 프로그래밍을 잘 몰라도 걱정 마세요! 이 가이드를 통해 쉽게 따라 할 수 있습니다.

---

## ✨ 주요 기능

*   **텍스트 기반 설계**: 폴더 구조와 파일 내용을 텍스트로 작성하여 직관적으로 관리합니다.
*   **시각적 비교 (Diff View)**: 스캐폴딩(파일 생성) 전후의 폴더 구조 변화를 한눈에 확인할 수 있습니다.
*   **안전한 작업**: 실수로 중요한 시스템 폴더를 건드리는 것을 방지하며, 기존 파일을 덮어쓰기 전에 경고 메시지를 표시합니다.
*   **단축키 지원**: 자주 사용하는 기능을 단축키로 빠르게 실행할 수 있으며, `Alt` 키를 누르면 화면에 단축키 힌트가 나타납니다.
*   **로그 기록**: 모든 작업 내역을 상세한 로그로 확인할 수 있어 문제 발생 시 원인 파악에 도움이 됩니다.

---

## 🚀 사용 방법

### 1단계: 애플리케이션 실행

`gui_app.py` 파일을 실행하여 "Tree Scaffolder" 프로그램을 시작합니다.

```bash
python gui_app.py
```

### 2단계: 파일 생성 위치 선택 (`1. Select Target Root Folder`)

스캐폴딩(폴더 및 파일 생성) 작업을 시작할 기준 폴더를 선택합니다.

1.  **`Browse...` 버튼 클릭**: 원하는 폴더를 찾아 선택합니다.
2.  **`Prev` 버튼 클릭**: 이전에 작업했던 폴더로 빠르게 돌아갑니다.
3.  **`Clear` 버튼 클릭**: 선택된 폴더 정보를 지우고 초기 상태로 되돌립니다.

> 💡 **주의**: 시스템의 중요한 폴더(예: `C:\Windows`, `Program Files` 등)는 실수로 선택되지 않도록 보호되어 있습니다. 안심하고 사용하세요.

### 3단계: 폴더 구조 설계 (`2. Define Scaffold Tree`)

왼쪽 상단의 'Scaffold Tree' 편집기에 원하는 폴더와 파일의 구조를 텍스트로 작성합니다.

*   **`@ROOT {{Root}}`**: 항상 이 문장으로 시작하며, 스캐폴딩의 최상위 폴더를 의미합니다. `{{Root}}` 부분은 실제 선택된 타겟 폴더 이름으로 대체됩니다.
*   **들여쓰기**: `Tab` 키나 4칸 공백으로 들여쓰기를 하여 폴더/파일의 계층 구조를 만듭니다.
*   **폴더와 파일 구분**:
    *   이름 끝에 `/` (슬래시)를 붙이면 **폴더**로 인식됩니다. (예: `MyFolder/`)
    *   이름 끝에 `/`가 없으면 **파일**로 인식됩니다. (예: `MyFile.txt`)

**예시:**

```
@ROOT {{Root}}

{{Root}}/
    ProjectX/
        Source/
            Core/
                Module.h
            Public/
        README.md
```

### 4단계: 파일 내용 작성 (`Source Code` 탭)

'Source Code' 탭 편집기에서 만들고 싶은 파일의 실제 내용을 작성합니다. 파일의 내용 시작과 끝은 `@@@FILE_BEGIN`과 `@@@FILE_END`라는 특별한 표시로 알려줘야 합니다.

*   **`@@@FILE_BEGIN [파일 경로]`**: 이 문장으로 파일 내용 작성을 시작합니다. `[파일 경로]` 부분에는 'Scaffold Tree'에서 정의한 파일의 경로를 `@ROOT`부터 시작하여 정확히 입력합니다.
*   **`@@@FILE_END`**: 이 문장으로 파일 내용 작성을 마칩니다.

**예시:** `ProjectX/README.md` 파일의 내용을 정의하는 방법:

```
@@@FILE_BEGIN {{Root}}/ProjectX/README.md
# 프로젝트 X 시작하기

이 프로젝트는 새로운 기능을 개발하기 위한 기본 뼈대를 제공합니다.
다음 단계를 따라 시작해 보세요:

1. 환경 설정
2. 빌드 실행
3. 테스트
@@@FILE_END
```

> 💡 **중요**: `@@@FILE_BEGIN`과 `@@@FILE_END` 사이의 모든 내용(줄바꿈, 공백 포함)은 작성한 그대로 파일에 저장됩니다.

> 💡 **빠른 트리 생성 팁: `make tree` 버튼**
> 'Source Code' 탭을 선택하면 에디터 위에 `make tree` 버튼이 활성화됩니다. 만약 `@@@FILE_BEGIN...END` 블록들이 이미 작성된 상태라면, 이 버튼을 클릭하여 해당 파일 경로들을 기반으로 'Scaffold Tree'의 구조를 자동으로 생성할 수 있습니다. 수동으로 트리 구조를 작성하는 과정을 단축시켜주는 편리한 기능입니다.

*   **`@@@COMMENT_BEGIN` / `@@@COMMENT_END`**: 이 태그들 사이에 있는 내용은 스캐폴딩 시 완전히 무시됩니다. 주석이나 임시 메모를 남길 때 활용할 수 있습니다.

### 5단계: 스캐폴드 계획 확인 (`Compute Diff` 버튼)

'Scaffold Tree'와 'Source Code' 편집기에 내용을 모두 작성했다면, `Compute Diff` 버튼을 눌러 스캐폴딩 계획을 확인합니다.

*   **`Before / After Diff` 탭**:
    *   **Before (Current State)**: 현재 선택된 타겟 폴더의 파일/폴더 구조를 보여줍니다.
    *   **After (Planned State)**: `Compute Diff`를 통해 예상되는 스캐폴딩 후의 파일/폴더 구조를 보여줍니다.
*   **색상으로 상태 확인**:
    *   **초록색**: 새로 생성될 파일/폴더.
    *   **파란색**: 기존 파일에 내용이 덮어쓰여질 예정인 파일.
    *   **빨간색**: 예상치 못한 충돌(예: 파일로 계획했지만 해당 경로에 이미 폴더가 있는 경우)이 발생한 항목. 빨간색이 있다면 스캐폴딩을 적용할 수 없습니다.

### 6단계: 스캐폴드 적용 (`Apply Scaffold` 버튼)

`After` 탭에서 계획된 구조에 문제가 없음을 확인했다면, `Apply Scaffold` 버튼을 눌러 실제 파일과 폴더를 생성합니다.

> ⚠️ **매우 중요**: 이 작업은 컴퓨터의 파일 시스템에 직접 변경사항을 적용합니다. 특히 파란색(`overwrite`)으로 표시된 파일은 기존 내용이 사라지고 새로운 내용으로 완전히 대체됩니다. **항상 중요한 파일은 백업**해두고, 신중하게 확인한 후에 진행하십시오.

*   **`Dry Run` 옵션**: `Compute Diff`를 누르기 전에 'Dry Run (don't write files)' 옵션을 체크하면, 실제 파일 생성 없이 계획만 시뮬레이션해볼 수 있습니다. 실제 적용 전에 꼭 활용하세요.
*   **`Open folder after Apply Scaffold` 옵션**: 스캐폴딩 완료 후, 생성된 폴더를 자동으로 열어줍니다.

### 7단계: 작업 로그 확인 (`Log` 탭)

모든 스캐폴딩 작업의 진행 상황과 오류, 경고 메시지는 'Log' 탭에 상세히 기록됩니다. 문제가 발생하면 이 로그를 확인하여 원인을 파악할 수 있습니다.

## 8. 단축키

애플리케이션 내의 다양한 기능을 키보드 단축키로 빠르게 실행할 수 있습니다.

*   **단축키 힌트 표시**: `Alt` 키를 누르고 있으면 현재 화면에 표시된 버튼 위에 해당 기능에 할당된 단축키가 노란색으로 나타납니다. `Alt` 키에서 손을 떼면 힌트가 사라집니다.
*   **단축키 목록**: 자세한 단축키 목록은 [기술 설명서(SCAFFOLDER_GUIDE.md)](SCAFFOLDER_GUIDE.md)를 참조하십시오.

---

## 9. 주의사항

*   **데이터 백업**: `Apply Scaffold` 기능은 기존 파일을 덮어쓸 수 있으므로, 중요한 데이터는 항상 미리 백업해두는 습관을 들여주세요.
*   **`Dry Run` 활용**: 실제 스캐폴딩 적용 전에는 반드시 `Dry Run` 옵션을 사용하여 모든 변경사항을 미리 확인하십시오.
