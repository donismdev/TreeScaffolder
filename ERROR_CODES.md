# TreeScaffolder Error Codes

This document lists the error codes generated by the TreeScaffolder V2 Multipatch engine. These codes are designed to help you quickly identify and resolve issues with your patch files.

## V2 Engine Errors (V2-xxx)

| Code | Name | Description | Solution |
| :--- | :--- | :--- | :--- |
| **V2-001** | `MISMATCHED_TAGS` | A `BEGIN` tag was closed by an `END` tag with a different keyword. | Ensure every `@@@KEYWORD_BEGIN` has a matching `@@@KEYWORD_END`. |
| **V2-002** | `UNCLOSED_BLOCK` | a `BEGIN` tag was never closed. | Check for missing `@@@END` tags at the end of your file. |
| **V2-003** | `UNEXPECTED_END` | Found an `END` tag without a corresponding `BEGIN` tag. | Check for duplicate `@@@END` tags or typos. |
| **V2-004** | `INVALID_NESTING` | A block is placed inside another block that doesn't support children. | Only `PATCH` blocks can contain children like `FIND`, `REPLACE`, etc. |
| **V2-010** | `TEXT_NOT_FOUND` | The text specified in a `FIND` block was not found in the target file. | Check the literal bytes in `runtime.log` (Hex/Diff). Look for extra spaces, tabs, or newlines. |
| **V2-011** | `AMBIGUOUS_MATCH` | Multiple occurrences of the `FIND` text were found. | Provide more context in your `FIND` block to make the target location unique. |
| **V2-012** | `MISSING_CONTEXT` | An operation (REPLACE, INSERT, etc.) was attempted without a preceding `FIND`. | Always place a `FIND` block immediately before the operation that depends on it. |
| **V2-020** | `SECURITY_TRAVERSAL` | A path contains `..` which attempts to escape the root directory. | Paths must only move downwards from `{{Root}}`. |
| **V2-021** | `INVALID_PATH_FORMAT` | A file path is missing the mandatory `{{Root}}` marker. | Every path in a V2 block must start with `{{Root}}`. |
| **V2-022** | `OUTSIDE_ROOT` | A path resolves to a location outside the targeted folder. | Ensure your paths are relative to the project root. |
| **V2-030** | `FILE_READ_FAILURE` | Could not read the target file from disk (permissions or encoding). | Ensure the file is valid UTF-8 and not locked by another process. |
| **V2-040** | `UNSUPPORTED_KEYWORD` | An unrecognized keyword was used in an `@@@` block (e.g., typo). | Check the `v2style` manual for supported keywords and check for typos. |

---
*For detailed byte-level debugging, check the `[DIAGNOSTIC]` entries in the latest session log.*
